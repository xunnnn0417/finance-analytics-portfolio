Process

Engineering Workflow

Step What I Did Why It Matters
Data ingestion Downloaded, converted, and checked market data before running tests. Bad data can make a strategy look better or worse than it is.
Model comparison Compared base setups with Hurst and HMM-style filters. A variant only matters if it improves behavior versus a baseline.
Stress testing Applied cost multipliers, Monte Carlo, walk-forward, and drawdown checks. Robustness matters more than one optimized equity curve.
Documentation Kept tables, charts, reports, and negative results. Research should be reviewable, not just impressive.

Conclusion

What I Learned

The useful part of this project is not a single return number. It is the workflow: collect data, validate it, define assumptions, compare variants, stress-test the results, and write down the limits. Strong backtests become candidates. Failed stress tests become useful filters. That is the kind of process I want my technical work to show.
