Process

Engineering Workflow

Step What I Did Why It Matters
Data ingestion 下載、轉換並檢查 market data，再開始跑測試。 資料有問題，backtest 結果就沒有意義。
Model comparison 比較 base setup、Hurst filter 和 HMM-style filter。 新的 variant 必須比 baseline 更有理由存在。
Stress testing 檢查 cost multipliers、Monte Carlo、walk-forward 和 drawdown。 Robustness 比單一最佳化 equity curve 更重要。
Documentation 保留 tables、charts、reports，也保留 negative results。 研究要能被 review，不只是看起來 impressive。

Conclusion

What I Learned

這個 project 的價值不是單一 return number，而是流程：collect data、validate it、 define assumptions、compare variants、stress-test results，最後把限制寫清楚。 Strong backtests 是 candidates，failed stress tests 也是有用的 filter。
